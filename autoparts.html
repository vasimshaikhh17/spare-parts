Sparraps Own Vendor Panel Flow

Features:
  • No subscription charge
  • Receives every request first
  • Maintains own prices & inventory
  • Uploads actual product images
  • Highest priority (configurable)
  • Operates through same order system
Flow:
Customer Request → Sparraps Own Vendor Panel → Check Inventory →
Available → Enter Own Price + Upload Image → Customer Approval → Order Fulfilled
Not Available → Reject or Timer Expires → Send to Subscribed Vendors

Customer & Vendor Confidentiality

Hidden from Customers:
  • Vendor name, phone, shop name
  • Vendor address & location
  • Vendor email, GST, bank details

Customer-facing barcode does not disclose vendor info

Hidden from Vendors (operationally):
  • Customer phone number
  • Customer email
  • Customer direct communication
  • Customer payment details

Logistics system receives only pickup/delivery information

20 Business rules

1. Only Sparraps Admin creates product listings.
2. Vendors cannot create public listings.
3. Vendors set own price for admin products.
4. Sparraps internal vendor has highest priority.
5. Request goes to internal vendor first.
6. If internal rejects, request goes to subscribed vendors.
7. Priority tiers: Platinum, Gold, Silver, Basic, Trial.
8. Admin configures timer & routing rules.
9. Customer & vendor never communicate directly.
10. Vendor identity hidden from customer.
11. Vendor uploads actual product photo after acceptance.
12. Images are background-removed & verified.
13. Payment after customer approves product option.
14. All payments to Sparraps via Razorpay.
15. Sparraps arranges pickup & delivery.
16. Unique barcode/QR per product.
17. 7-day return period from delivery.
18. Return rejected if barcode missing/damaged.
19. Vendor settlement frequency configurable.
20. All settings manageable via admin panel.
